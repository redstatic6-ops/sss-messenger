# Добавление функции звонков в SSS Messenger

## ✅ Выполнено

Функция звонков успешно добавлена в мессенджер!

### Что было сделано:

1. **Установлена библиотека simple-peer** для WebRTC соединений
2. **Создана SQL миграция** `add_calls_support.sql` с таблицами:
   - `calls` - история звонков
   - `call_signals` - временные WebRTC сигналы
3. **Создан хук `useWebRTC.js`** для управления звонками через WebRTC
4. **Созданы UI компоненты:**
   - `IncomingCallModal.jsx` - модальное окно входящего звонка
   - `ActiveCallWindow.jsx` - полноэкранное окно активного звонка
5. **Обновлен `ChatWindow.jsx`** - добавлены кнопки аудио/видео звонков в хедер чата
6. **Сборка проверена** - приложение собирается успешно

### Функционал:

- 🎧 **Аудиозвонки** - голосовая связь между пользователями
- 📹 **Видеозвонки** - видеосвязь с возможностью отключения камеры
- 🔇 **Управление микрофоном** - включение/выключение во время звонка
- 📷 **Управление камерой** - включение/выключение видео
- ⏱️ **Таймер звонка** - отображение длительности
- 🔔 **Входящие звонки** - уведомление с возможностью принять/отклонить
- 📊 **История звонков** - сохранение в БД

### Технологии:

- **WebRTC** - P2P аудио/видео соединения
- **simple-peer** - упрощенная работа с WebRTC
- **Supabase Realtime** - signaling сервер для установки соединения
- **STUN серверы Google** - для работы через NAT

## 🚀 Следующие шаги:


### 3. Протестировать звонки:

- Открыть чат с другом
- Нажать кнопку 🎧 (аудиозвонок) или 📹 (видеозвонок)
- Разрешить доступ к микрофону/камере
- Второй пользователь получит входящий звонок

## ⚙️ Настройка для production:

### TURN сервер (опционально):

Для гарантированного соединения через сложные NAT добавить TURN сервер в `src/hooks/useWebRTC.js:145`:

```javascript
config: {
  iceServers: [
    { urls: 'stun:stun.l.google.com:19302' },
    {
      urls: 'turn:your-turn-server.com:3478',
      username: 'username',
      credential: 'password'
    }
  ]
}
```

### Разрешения в Electron:

Файл `electron/main.js` уже должен разрешать доступ к медиа-устройствам. Если нет, добавить:

```javascript
session.defaultSession.setPermissionRequestHandler((webContents, permission, callback) => {
  if (permission === 'media') {
    callback(true);
  }
});
```

## 📝 Примечания:

- Звонки работают только между двумя пользователями (P2P)
- Требуется HTTPS или localhost для работы WebRTC
- Браузер запросит разрешение на доступ к микрофону/камере
- STUN серверы Google бесплатны, но не гарантируют соединение через все NAT
